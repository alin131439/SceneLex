# 雅思单词背诵工具 — 技术架构文档

## 1. 技术选型

### 1.1 选型原则
- Demo 阶段追求**开发效率**与**视觉表现力**，不引入重型框架
- 纯前端方案，无需后端服务即可展示完整交互原型
- 优先选择社区活跃、文档完善的方案

### 1.2 技术栈

| 层 | 技术 | 版本 | 选型理由 |
|---|------|------|----------|
| 框架 | React | 18.x | 组件化开发，生态丰富 |
| 构建工具 | Vite | 5.x | 极速 HMR，零配置启动 |
| 样式方案 | CSS Modules + CSS Variables | — | 轻量级，无运行时开销 |
| 动画引擎 | Framer Motion | 11.x | 手势动画支持优秀，卡片拖拽首选 |
| 路由 | React Router | 6.x | SPA 页面切换 |
| 图标 | Lucide React | — | 轻量图标库 |
| 静态数据 | 内联 JSON | — | Demo 无需后端 |
| 部署 | Vite Preview / Vercel | — | 静态站点一键部署 |

### 1.3 未选方案说明

| 未选方案 | 未选原因 |
|----------|----------|
| Vue / Nuxt | 团队 React 经验更丰富；Framer Motion 与 React 生态契合度更高 |
| Next.js | Demo 无 SSR 需求，Vite 更轻量 |
| Tailwind CSS | Demo UI 定制化程度高，CSS Modules 更灵活 |
| GSAP | 学习曲线陡，Framer Motion 声明式 API 更适合快速原型 |
| Taro / uni-app | 产品形态确定为网页，无需跨端编译 |

## 3. 核心模块设计

### 3.1 路由设计

```
/           →  Home       首页（学习概览）
/learn      →  Learn      学习页（卡片滑动）
/stats      →  Stats      统计页（数据展示）
```

使用 React Router v6 的 `createBrowserRouter` 实现。

### 3.2 卡片滑动引擎（CardStack）

这是整个 Demo 最核心的交互模块。

**技术方案**：基于 Framer Motion 的 `useDrag` + `useAnimation` 实现

**交互状态机**：

```
                   ┌─────────┐
        loadNext() │  idle   │ dragStart
              ┌───►│ (静止)   │◄────┐
              │    └────┬────┘     │
              │         │          │
              │    drag │          │ drag (跟手)
              │         ▼          │
              │    ┌─────────┐     │
              │    │dragging │─────┘
              │    │(拖拽中)  │
              │    └────┬────┘
              │         │ dragEnd
              │         ▼
              │    ┌─────────┐
              │    │deciding │  判断偏移量
              │    │(判定中)  │
              │    └────┬────┘
              │     ┌───┼───┐
              │     │   │   │
              │  left│none│right
              │     ▼   ▼   ▼
              │   swipe  回弹  swipe
              │   left        right
              │     │   │   │
              │     └───┼───┘
              │         │
              └─────────┘
```

**判定逻辑**：
- 水平偏移 > 100px 且速度 > 500 → 触发滑动
- 水平偏移 > 卡片宽度的 30% → 触发滑动
- 否则 → 回弹到原位

**动画参数**：
```js
const SWIPE_THRESHOLD = 0.3;    // 滑动触发阈值（相对于卡片宽度）
const VELOCITY_THRESHOLD = 500; // 速度阈值 (px/s)
const SPRING_CONFIG = {         // 回弹弹簧参数
  type: "spring",
  stiffness: 300,
  damping: 25,
};
const EXIT_DURATION = 0.3;     // 飞出动画时长 (s)
```

## 5. 动画清单

| 动画 | 触发条件 | 实现方式 | 时长 |
|------|----------|----------|------|
| 卡片入场 | 新卡片出现 | `initial={{ scale: 0.8, opacity: 0 }}` → spring | 400ms |
| 卡片拖拽跟手 | 手指/鼠标拖拽 | `useDrag` 实时更新 x/y | 实时 |
| 卡片飞出（认识） | 左滑超阈值 | `animate={{ x: -500, rotate: -15, opacity: 0 }}` | 300ms |
| 卡片飞出（不认识）| 右滑超阈值 | `animate={{ x: 500, rotate: 15, opacity: 0 }}` | 300ms |
| 卡片回弹 | 未达阈值释放 | `animate={{ x: 0, y: 0 }}` spring | 400ms |
| 卡片堆叠视觉 | 始终可见 | 下层卡片 `scale(0.95)` + `translateY(-10px)` | — |
| 进度条填充 | 学习完成时 | CSS transition `width` | 600ms |
| 按钮点击 | 点击瞬间 | `whileTap={{ scale: 0.96 }}` | 100ms |
| 页面切换 | 路由变化 | `AnimatePresence` fade | 300ms |
| 数字跳动 | 统计数据更新 | `useSpring` 数值动画 | 800ms |

## 6. 响应式策略

Demo 以移动端为第一优先，同时兼容桌面端展示：

```
移动端 (320px - 768px)     → 完整交互体验
平板端 (768px - 1024px)    → 卡片最大宽度 420px，居中
桌面端 (1024px+)           → 卡片最大宽度 480px，居中，展示键盘提示
```

实现方式：
- 使用 CSS `max-width` + `margin: auto` 限制内容宽度
- 使用 `min()` / `clamp()` 实现流体字号
- 拖拽同时支持 touch 和 mouse 事件（Framer Motion 原生支持）

## 7. 性能优化

| 优化项 | 策略 |
|--------|------|
| 卡片复用 | 卡片池维护 3 张卡片，循环复用，避免频繁创建销毁 |
| 动画卸载 | 飞出卡片在动画完成后从 DOM 移除 |
| will-change | 拖拽中的卡片添加 `will-change: transform` |
| 图片资源 | Demo 无图片，纯 CSS + SVG 图标，零网络请求 |
| 懒加载 | 统计页图表组件通过 `React.lazy` 按需加载 |
